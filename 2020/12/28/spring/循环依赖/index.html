spring循环依赖

发表于 2020-12-28 | | 热度: ℃

有两个bean a 中需要注入bean ，b 中注入 a

开始启动测试

  • singletonObjects：单例缓存池 创建好的单例bean 会放到这里面
  • earlySingletonObjects 二级缓存池
  • earlySingletonObjects 三级缓存池

Object sharedInstance = getSingleton(beanName);

第一次走该方法时，sharedInstance 返回null ，毕竟一级 二级 三级缓存都没有缓存 a bean 的数据

当调用getObject 方法的时候，走createBean(beanName, mbd, args); 方法

走Object beanInstance = doCreateBean(beanName, mbdToUse, args); 方法

​ instanceWrapper = createBeanInstance(beanName, mbd, args);内部是通过反射创建对象。属性都还没有赋值

讲上面创建的bean 实例 对象放到三级缓存中

1、当走到这边，会有一个AutowiredAnnotationBeanPostProcessor 处理器，用来处理Autowired 注入。

走descriptor.resolveCandidate(autowiredBeanName, type, this);

走到这边，又会调用 beanFactory.getBean(beanName); 因为我们a bean 主要注入b bean属性

Object sharedInstance = getSingleton(beanName); 这个时候 获取b bean 同样返回null,和上面 bean a 一样 走到doCreateBean

通a 一样，会通过反射创建b的对象，然后将b 放到三级缓存中

然后走populateBean(beanName, mbd, instanceWrapper); 属性注入 ，这个时候会发现 b 中需要注入bean 对象 a ,然后又走到

这个时候在三级缓存中查找到bean a , 然后将bean 放到二级缓存中，从三级缓存中移除

这个时候b bean 对象调用singletonObject = singletonFactory.getObject(); 完成，newSingleton = true; 会走下面方法

然后一直返回，讲获取到的bean 对象返回给 a 对象进行属性赋值

这个里面只会从一级缓存里面查询出来，
